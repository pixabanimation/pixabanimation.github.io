⚙️ Technical SEO ⚠️ Needs Work

  • ✓
    robots.txt
    Properly configured. Allows crawling, disallows admin/auth/checkout pages, points to sitemap.
  • ✓
    XML Sitemap
    Exists at /sitemap.xml with all major pages. Includes lastmod dates.
  • ❌
    SPA Hash Routing
    CRITICAL: All internal pages use hash-based URLs (#/shop, #/about, etc.). Google can index some JavaScript-rendered SPAs, but hash-based routing is NOT reliably crawlable. Most of your pages may not appear in search results.
    💡 Migrate to a static site generator (e.g., 11ty, Hugo, Astro) or use History API-based routing with prerendering. For a GitHub Pages site, consider 11ty or Astro to pre-render all pages as static HTML.
  • ❌
    Sitemap Contains Hash URLs
    The sitemap includes URLs with #/ fragments (e.g., /sitemap.xml has #/shop, #/about). URL fragments (#) are not sent to servers in HTTP requests — Google may not properly crawl these URLs.
    💡 Sitemap URLs should use clean paths like /shop, /about. Requires migrating away from hash routing.
  • ❌
    404 Page
    The 404.html page uses JavaScript sessionStorage redirect to the homepage. While creative, this means server-side 404s silently redirect — search engines won't see proper 404 status codes and may not handle errors correctly.
    💡 Provide a proper 404 page with navigation and a link back to the homepage. For GitHub Pages, ensure the 404.html includes useful content.
  • ✓
    HTTPS
    GitHub Pages serves over HTTPS automatically.
